FAQ по разделам админки

User - пользователи веб приложения, внутри:

Username - логин,имя пользователя в системе, в его качестве используется e-mail

Fio - Фамилия, имя и отчество пользователя, указанные при регистрации

Company - компания, в которой работает пользователь

Position - должность пользователя в указанной компании

Date_of_birth - дата рождения

Phone number - номер телефона

Role - роль: пользователь(user) или админ(admin)

При открытии для редактирования или создания, у пользователя(User) также будут два поля:

Courses - показывает, на какие курсы записался пользователь

User Answer - показывает, на какие вопросы ответил пользователь и как

Course - курсы, информация о них:

Name - название курса(будет отражаться на главной странице)

Info - информация о курсе(будет отражаться на главной странице)

Conditions - условия для поступления курс, цена и пр.(будет отражаться на главной странице)

Content - содержание курса(будет отражаться после поступления пользователем на курс при переходе непосредственно в курс для занятий)

При открытии для редактирования или создания, также появляется поле Lessons - показывает, какие уроки входят в этот курс, а также позволяет добавлять или удалять уроки.

Lesson - уроки, которые могут быть добавлны в курсы:

Name - название урока

Material Type - на данный момент нужен лишь для того, чтобы показать, будет ли использоваться в материале элемент bootstrap carousel(листалка слайдов или изображений). Если он используется - указать slides, если нет - указать noslides.

Material - материал урока. В базе данных хранится с html-тегами, поэтому на данный момент для добавления элементов с других сайтов стоит сохранять непосредственно embed или iframe-ссылку, в тех элементах, в которых требуется непосредственное редактирование текста, встроен редактор

Questions to pass - указывает, на сколько вопросов необходимо ответить пользователю в конце урока, чтобы он считался пройденным в статистике его профиля.

При открытии для редактирования или создания, также будут видны два поля:

Questions - позволяет выбрать и поставить в урок заранее созданный вопрос. В урок можно вставлять вопросы разных типов(открытые(open) - требуется вписать вариант ответа; и закрытые(closed) - требуется выбрать вариант ответа из предложенных)

Slides - выбор слайдов для отображения, если вы выбрали Material Type - slides. Слайды хранятся локально на сервере.

User_answer - здесь хранятся ответы пользователей на вопросы в конце урока:

Users - пользователи

Questions - вопрос, на который был дан ответ

Lesson Id - Id урока, в котором есть этот вопрос

Lesson Name - название урока, в котором есть этот вопрос

User Answer - ответ пользователя

Answer Status - статус ответа пользователя, может быть correct или wrong

Question - показывает вопросы, которые можно вставить в уроки:

Correctanswer - правильный ответ на поставленный вопрос

Question Text - текст вопроса

Question Type - тип вопроса(open - открытый, closed - закрытый)

При открытии для редактирования или создания также становятся видны два поля:

Answervariants - варианты ответа для закрытых вопросов тестового типа, создаются отдельно в соответствующем разделе с аналогичным названием, можно выбрать любой к любому вопросу. Если вопрос открытого типа - оставлять пустым.

User Answers - можно увидеть какой пользователь и как ответил на данный вопрос, при необходимости удалить его ответ.


Answer Variant - раздел с вариантами ответа на тестовые вопросы:

Answer Text - вариант ответа на вопрос. Редактируется только это поле.
